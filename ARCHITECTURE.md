# 冥刻翻訳碑 (Meikoku Shift) - 内部構造と管理ガイド

このプロジェクトは、高い変換精度とオフラインの堅牢性を両立させるためのハイブリッド・エンジン構成をとっています。

## 1. 変換エンジンの二重構造 (Hybrid Engines)

`src/services/conversionService.ts` が、状況に応じて最適なエンジンを切り替えます。

| モード | エンジン | 特徴 | 発動条件 |
| :--- | :--- | :--- | :--- |
| **Celestial** | Gemini API | 文脈を考慮した極めて高い変換精度 | APIキー設定済み かつ ネットワークあり |
| **Terrestrial** | Kuroshiro (Kuromoji) | 高速・軽量・オフライン対応 | APIキーなし または ネットワーク切断時 |
| **None** | Wanakana | 単純なかな変換のみ | 両方のエンジンが利用不能な場合 |

## 2. 辞書管理とビルドフロー

リポジトリサイズを抑えつつ、オフライン機能を維持するための工夫です。

- **辞書データの配置**: `npm install` 時に取得される `kuromoji` の辞書ファイルを、`package.json` の `prebuild` スクリプトで `public/dict/` に自動コピーします。
- **Git管理**: `public/dict/` は巨大なバイナリデータを含むため、`.gitignore` で追跡対象から除外しています。デプロイ時は GitHub Actions 上で再生成されます。
- **GitHub Pages**: `vite.config.ts` の `base` パス設定により、サブディレクトリ環境でのデプロイをサポートしています。

## 3. 開発時のトラブルシューティング

- **Viteのキャッシュエラー**: ライブラリの入れ替え後にエラーが出る場合は、以下のコマンドでキャッシュを強制クリアして起動してください。
  ```bash
  npm run dev -- --force
  ```

## 4. 今後の拡張性

- `conversionService.ts` を拡張することで、新たなエンジン（例：Sudachiが再安定化した際など）の追加が容易な設計になっています。
- UIとロジックを厳密に分離しているため、デザインの変更がロジックに影響を与えません。
